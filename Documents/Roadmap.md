# Roadmap

## Текущее состояние

- **Продукт:** ORDINA Svelte — PWA на Svelte 4 + TypeScript с полной функциональной совместимостью с ORDINA 1 (vanilla JS). Фичи: финансы (расходы, долги, повторяющиеся), задачи (дневные/месячные/годовые с carry-over), календарь, дашборд с виджетами, i18n (EN/RU/AZ/IT), темы, PWA.
- **Прогресс:** Чек-лист парности с ORDINA 1 ведётся в `docs/PROGRESS.md` (автогенерация через `npm run progress`). Тесты (unit, e2e, visual, a11y) присутствуют; часть E2E может быть помечена skip до настройки тестового Firebase и данных.

## Приоритеты развития (допущения)

1. **Стабилизация парности:** Закрытие оставшихся пунктов в `docs/PROGRESS.md`, прохождение всех критичных E2E без skip, финальная проверка на одних данных с ORDINA 1.
2. **Качество:** Поддержание WCAG 2.1 AA, стабильность визуальных и property-based тестов, покрытие критичных путей unit-тестами.
3. **Деплой:** Production-сборка с base path для целевой платформы (например GitHub Pages `/ORDINA.github.io/`), проверка работы SW и офлайн-сценариев.
4. **Дальнейшие улучшения (по возможности):** Firebase Emulator для изолированных E2E; централизованная отправка ошибок (e.g. Sentry) в production; оптимизация загрузки (lazy-загрузка табов/модалок при необходимости).

## Ограничения и допущения

- Схема Firestore и набор фич синхронизированы с ORDINA 1; изменения схемы требуют согласования.
- Roadmap не фиксирует жёсткие сроки; документ служит ориентиром для планирования и для AI при оценке «что можно улучшить дальше» без нарушения правил из `Skeleton.md`.
