# User Guide

Краткое руководство по использованию приложения. Детали интерфейса — в `UX_UI_Logic.md`.

## Запуск и вход

1. **Установка:** `npm install` в корне проекта.
2. **Запуск:** `npm run dev` — приложение доступно по адресу из вывода (по умолчанию порт 3002).
3. **Вход:** на экране входа доступны вход по email/паролю, регистрация и вход через Google. Учётные данные — Firebase Authentication того проекта, который задан в `.env` (см. README и `Security_and_Data.md`).

## Навигация

- **Вкладки (сайдбар):** Dashboard, Expenses, Debts, Recurring, Tasks, Calendar. Выбор вкладки переключает основной контент.
- **Месяц:** на вкладках с месячными данными (Expenses, Recurring, Tasks) выбор месяца — через MonthSelector; данные подгружаются для выбранного месяца.
- **Модалки:** добавление/редактирование сущностей (расход, долг, платёж, повторяющийся расход, категория, задачи, событие календаря, калькулятор, список покупок, настройки, профиль) открываются кнопками или PWA-shortcuts и закрываются кнопкой закрытия или по области вне модалки (где предусмотрено).

## Основные сценарии

### Расходы
- Вкладка **Expenses** → кнопка добавления расхода → форма (название, категория, сумма, дата). Редактирование и удаление — из строки таблицы. Экспорт в CSV — через соответствующую кнопку. Валюта отображения переключается в Header (AZN/USD).

### Долги
- Вкладка **Debts** → добавление долга (название, общая сумма, комментарий). Платежи по долгу — через модалку «Платёж»; сумма долга уменьшается на сумму платежа. Редактирование/удаление — из таблицы. Экспорт в CSV доступен.

### Повторяющиеся расходы
- Вкладка **Recurring** → шаблоны повторяющихся расходов (название, сумма, день месяца). По каждому месяцу отображается статус (оплачено/ожидает); статус можно обновлять. Шаблоны создаются/редактируются через модалки.

### Задачи
- Вкладка **Tasks** — три блока: дневные (на выбранную дату), месячные (на выбранный месяц), годовые (на текущий год). Добавление — через соответствующие модалки. Статусы: не выполнено / выполнено / пропущено. Невыполненные дневные задачи переносятся на следующий день в полночь (или при возврате в приложение); месячные — на следующий месяц.

### Календарь
- Вкладка **Calendar** → сетка месяца. Добавление события — по дате или кнопке; типы: event, birthday, meeting, wedding. Редактирование и удаление — через модалку события.

### Дашборд
- Сводные карточки по финансам и задачам, часы, виджеты погоды, новостей и радио. Погода и новости зависят от внешних API и ключей (см. README); при их отсутствии виджеты могут быть пустыми.

## Настройки и профиль

- **Профиль:** имя, аватар (если поддерживается провайдером) — через модалку профиля из Header.
- **Настройки:** тема (светлая/тёмная), язык (EN/RU/AZ/IT), при необходимости другие опции — через модалку настроек. Тема и язык сохраняются в localStorage.

## Офлайн

- При потере сети отображается индикатор офлайн. Часть данных может быть доступна из кэша; запись изменений ставится в очередь и синхронизируется при восстановлении связи. Ограничения офлайн-режима описаны в E2E-тестах (`tests/e2e/offline.spec.ts`) и в `Quality_and_Maintenance.md`.

## PWA и установка

- В production приложение можно установить как PWA (иконка «Установить» в браузере). Shortcuts в манифесте могут открывать приложение с действием (например, добавление расхода или задачи); параметры обрабатываются при старте (см. `App.svelte`, `handlePWAShortcuts`).
