# Product Philosophy

## Принципы продукта

1. **Один источник правды по данным** — состояние приложения в Svelte stores; данные из Firestore приходят через сервисы и listeners, без дублирования логики в компонентах.
2. **Совместимость с ORDINA 1** — та же схема Firestore и набор фич; пользователь может работать с одними данными в vanilla- и Svelte-версии.
3. **Офлайн-первый** — приложение устанавливается как PWA; при отсутствии сети показывается индикатор и по возможности используются кэш и отложенная синхронизация.
4. **Доступность** — ориентация на WCAG 2.1 AA: клавиатурная навигация, контраст, семантика, полифиллы при необходимости.
5. **Локализация** — интерфейс только через i18n; никаких захардкоженных строк для пользователя.

## Ценности

- **Предсказуемость:** понятная навигация (вкладки + модалки), единые паттерны ввода (формы в модалках).
- **Производительность:** виртуальный скролл для длинных списков, разбиение бандла (Firebase в отдельный chunk), минификация и отключение console в production.
- **Надёжность:** глобальный обработчик ошибок, ErrorBoundary, логирование; тесты на критические сценарии.

## Допущения

- Пользователь имеет аккаунт Firebase (email или Google) и согласен на хранение данных в Firestore.
- Курс AZN/USD задаётся в приложении (фиксированный); погода/новости/радио зависят от внешних API и при их недоступности виджеты могут быть пустыми или отключёнными.
- Carry-over задач выполняется при смене дня/месяца (полночь или возврат на вкладку); длительное отсутствие в приложении может потребовать однократной проверки при следующем входе.
- Production-деплой предполагает base path `/ORDINA.github.io/` (GitHub Pages); при другом деплое base и путь SW нужно согласовать.
