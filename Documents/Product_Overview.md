# Product Overview

## Продукт

**ORDINA Svelte** — PWA (Life Order Assistant) для управления личными финансами и жизненными задачами. Реализация на Svelte 4 + TypeScript с полной функциональной совместимостью с ORDINA 1 (vanilla JS); общий Firebase-проект и схема Firestore.

## Целевая аудитория

Пользователи, которым нужен единый инструмент для:
- учёта расходов и долгов по месяцам;
- повторяющихся платежей и категорий;
- дневных, месячных и годовых задач с переносом незавершённых;
- календаря событий (события, дни рождения, встречи, свадьбы);
- быстрого дашборда с итогами, погодой, новостями, радио.

Ориентация на личное использование; мультиязычность (EN/RU/AZ/IT) и валюта AZN/USD.

## Ключевые возможности

| Область | Возможности |
|---------|--------------|
| **Финансы** | Расходы по месяцам, долги с платежами, повторяющиеся расходы, категории, экспорт в CSV, переключение AZN/USD с фиксированным курсом. |
| **Задачи** | Дневные/месячные/годовые задачи, статусы (не выполнено / выполнено / пропущено), автоматический перенос (carry-over) при смене дня/месяца. |
| **Календарь** | События с типами (event, birthday, meeting, wedding), привязка к дате, просмотр по месяцам. |
| **Дашборд** | Сводные карточки, часы, погода (OpenWeatherMap), новости (RSS), радио (Zeno.FM) с эквалайзером. |
| **Приложение** | Вход по email и Google, светлая/тёмная тема, язык EN/RU/AZ/IT, офлайн-режим и установка как PWA. |

## Границы продукта

- **В рамках:** один пользователь на аккаунт Firebase; данные в Firestore; клиентское приложение (SPA); офлайн-чтение/очередь записей через SW.
- **Вне рамок:** многопользовательские сценарии, биллинг/подписки в приложении, свой backend кроме Firebase, миграция данных из других форматов (кроме экспорта CSV).

## Технологический контекст

- **Фреймворк:** Svelte 4 (не SvelteKit).
- **Сборка:** Vite 5.
- **Стили:** Tailwind CSS 3 + кастомный CSS.
- **Бэкенд:** только Firebase (Auth + Firestore).
- **Тесты:** Vitest (unit/property), Playwright (e2e), визуальные и a11y (axe).
